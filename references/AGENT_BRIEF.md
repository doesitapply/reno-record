## 1. What This Is

**The Reno Record** is a public legal accountability archive documenting *State v. Cameron Doyle Church*, CR23-0657, Second Judicial District Court, Washoe County, Nevada. It is simultaneously:

1. A **public-facing website** at `therenorecord.manus.space` — a narrative archive for journalists, attorneys, advocates, and the general public
2. A **legal document management system** — ingestion, classification, versioning, violation tagging, predicate analysis
3. A **civil rights accountability tool** — pattern analysis, actor tracking, judicial comparison, missing predicate detection
4. A **personal legal defense infrastructure** — built and operated by the defendant, Cameron Doyle Church, who is representing himself pro se

The operator (Cameron) is the subject of the case. He built this system while under a no-bail bench warrant, then while in custody, then while the case was stayed. Every feature has a real-world legal consequence. Nothing here is hypothetical.

### The Comparator: Richard Joyce
Joyce stabbed Church on April 10, 2023. Case RCR2023-122783. Bail reduced from $150,000 to $500. Charges dismissed at preliminary hearing. Prosecutor: Amos Stege (Chief DDA). Stege later intervened in Church's case on September 26, 2025 to file a "threat" notice based on a medical email.

### The "Threats"
DA Merchant's April 2025 no-bail motion characterizes Church's 31 emails as "insulting, threatening." The emails are in the record as Exhibit 1. Their content: Brady v. Maryland citations, NRS 239.0107 references, Valdez-Jimenez v. Eighth Jud. Dist. Ct. citations, Eighth Amendment arguments, public records requests. One email alleges Church's phone — seized at arrest — contained exculpatory video that was subsequently destroyed (Brady violation). No response to this allegation appears in the record.

### The Operation
From DA Merchant's own April 18, 2025 no-bail motion: *"law enforcement to include the federal US Marshall's Service, worked diligently to affect his capture with the use of a tracker on a vehicle associated with him and worked for weeks to pattern his movements. The first attempt to arrest him was unsuccessful, with the Defendant barricading himself in his house for days and refusing to come out. He was not captured willingly, but rather through a ruse that got him to eventually exit his home."* This describes resources deployed for a non-violent property dispute bench warrant.

### The Competency Proceedings
- No medical evidence was presented at the December 5, 2024 Young hearing
- Referral basis (per subsequent evaluation report): Church was "uncooperative and abusive to counsel, staff, and everyone at the court"
- Both evaluators employed by NaphCare (private jail medical contractor)

## 7. Core Systems

### Docket Goblin
AI-powered document ingestion pipeline. Drop a PDF → extracts text → identifies dates, actors, case numbers → classifies by record type (state case / federal case / supporting / unfiled) → tags violations → creates timeline event draft → QC supervisor reviews → admin approves. Powered by `invokeLLM`. Includes a chat interface for document analysis.

Key procedures: `docketGoblin.ingest`, `docketGoblin.draftForDocument`, `docketGoblin.applyDraftToDocument`, `docketGoblin.send`

### Evidence Engine
Document classification and versioning system. Every document has: `recordStatus` (on_record_state / on_record_federal / supporting / unfiled_not_on_record / unclassified), `reviewStatus` (pending / approved / rejected), `publicStatus` (boolean), `aiPolicy` (no_ai_processing / ai_allowed), full version history in `documentVersions`.

Key procedures: `evidenceEngine.batchClassify`, `evidenceEngine.reclassify`, `evidenceEngine.versions`, `evidenceEngine.restoreVersion`

### Missing Predicate Report
Analyzes all timeline events against the document archive to identify court actions without locatable supporting predicates. Uses LLM batch analysis (10 events/batch). Classifies each event as: Located / Partial / Contradicted / Not Located / Off-Record / Needs Human Review. Generates severity scores (liberty impact / counsel authority / procedural disposition / administrative). Exports to CSV and print/PDF. Court-safe language enforced throughout.

Key procedures: `predicate.generateReport`, `predicate.getReport`, `predicate.getReportStats`

### Violation Tag System
15 violation tag types, 137 total tags across 50+ documents. Tags include: Brady violations, Faretta violations, speedy trial, ineffective assistance, due process, judicial misconduct, prosecutorial misconduct, competency procedure violations, etc. Each tag has a source quote and citation.

### Judicial Pattern Audit
Compares the Breslow/CR23-0657 docket against comparable cases in the same court. Tracks: trial held, continuances, counsel changes, Faretta handling, competency raised, discovery issues, warrant use. Generates a comparative metrics report.

### Public API
REST API with OpenAPI spec and MCP manifest. Queryable by external tools, AI agents, and attorneys. API key management via `apiKey.*` procedures.

### Billing / Subscriptions
Stripe integration (test environment configured). Tiers: free, receipts, goblin_pro. Credit packs for Goblin usage. Billing portal via Stripe.

## 9. Key Constraints and Operational Notes

### What must never happen
- Do not generate legal conclusions. All findings must be framed as record-integrity observations: "Reviewed record does not locate…", "Predicate not found in reviewed materials…"
- Do not fabricate, seed, or mock case facts, violation tags, or document content
- Do not store file bytes in the database — use S3 via `storagePut`
- Do not call LLM from frontend — server-side only via `invokeLLM`
- Do not hardcode port numbers in server code

### Court-safe language requirements
Every AI-generated output in this system must use court-safe framing:
- "The reviewed record does not locate a supporting predicate for…"
- "Predicate not found in reviewed materials…"
- "Supporting authority appears partial…"
- "The record should identify…"
- Never: "proves," "demonstrates," "shows misconduct," "illegal," "criminal"

### Document classification logic
- `on_record_state` — filed in CR23-0657 state court
- `on_record_federal` — filed in 3:24-cv-00579 federal court
- `supporting` — supporting/reference documents (not filed)
- `unfiled_not_on_record` — exists but not in the official record
- `unclassified` — pending Goblin classification (26 documents as of last count)

### Access control
- Public: browse documents, timeline, actors, patterns, search, case report, predicate report (read-only)
- Authenticated user: submit stories, file PRRs, use Goblin (free tier), request audits
- Admin: all CRUD, Goblin ingest, batch classify, violation tag management, user management, predicate report generation

## 10. Strategic Context

### Why this system exists
Cameron Church built this system because:
1. Five attorneys cycled through his case without filing a single substantive motion
2. Every pro se filing he submitted was struck with a rubber stamp ("DFX: DEFENDANT REPRESENTED BY COUNSEL")
3. A competency evaluation was ordered — based on the quality of his legal arguments, not medical evidence — effectively silencing him
4. The case has been pending for 1,216+ days with no trial
5. A federal civil rights lawsuit naming the judge and prosecutor is pending, and the state court has not addressed this conflict

The archive is his primary mechanism for: (a) preserving the record, (b) creating public accountability, (c) supporting the federal case, and (d) demonstrating that his pro se legal work is substantive, cited, and coherent — directly contradicting the competency referral basis.

### What makes this legally significant
- The DA's own motion describes the defense theory as a "rent dispute" — corroborating the defense
- The DA's own exhibit (31 emails) shows legal arguments, not threats
- Both competency evaluators work for the same private contractor and reached opposite conclusions
- The second evaluator's report explicitly links the competency referral to Church's desire to represent himself
- A recorded call confirms law enforcement knew Church's location 55 days before the arrest
- The prosecutor is a named defendant in a pending federal civil rights case
- The judge is a named defendant in the same federal case
- The case is now stayed and a disqualification question has been referred

### What this system is not
- It is not a legal filing system (documents are archived, not e-filed)
- It is not a case management system for attorneys
- It is not a news publication (no editorial opinion — only record-sourced observations)
- It is not a threat or harassment tool (all content is sourced to public records)

## 13. How to Operate This System

### Adding a new document
1. Go to `/admin-hub` → Documents tab → Upload
2. Or use Docket Goblin: drop PDF → `docketGoblin.ingest` → review draft → `docketGoblin.approveIngest`
3. Assign violation tags via Admin Hub or Admin panel
4. Link to timeline events via `actorLink.addDocumentLink` or Admin Hub

### Running the Missing Predicate Report
1. Log in as admin → `/missing-predicate` → click "Generate Report"
2. LLM analyzes all timeline events in batches of 10
3. Results appear in the table with severity scores and recommended requests
4. Export to CSV or print for court use

### Adding a timeline event
1. `/admin-hub` → Timeline tab → "Add Event"
2. Or via `timeline.adminCreate` procedure
3. Link source documents via `sourceDocuments` field (JSON array of document IDs)

### Promoting a user to admin
Execute SQL: `UPDATE users SET role = 'admin' WHERE email = 'user@example.com';`

### Checking for undated documents
`/admin-hub` → Documents tab → filter by `recordStatus = 'unclassified'` → use Batch Classify
